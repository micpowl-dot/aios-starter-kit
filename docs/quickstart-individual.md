# Quickstart — solo operator

The fastest path from clone to "the system answers questions better than I would."

## Day 1 (~20 min)
1. Clone the repo, open it in Claude Code.
2. Paste `SETUP-PROMPT.md`. It runs `/onboard` (7 questions, ~15 min). Paste a real writing sample,
   don't describe it.
3. At the end you have: filled `context/` and `CLAUDE.md`, a `references/voice.md`, a populated
   `connections.md`, and a vault pointed at your notes.

## Week 1
- Bring real questions to the AIOS instead of opening six tabs. Make real decisions; log them to
  `decisions/log.md` (or let `/level-up` capture them).
- Wire your first **connection** (the one tool you check most). MCP if there is one, else a script or
  an export. Add a row to `connections.md`.

## Day 7
- Run `/audit`. Read the Four Cs gap report. Close the single highest gap (usually a missing connection).

## Day 14
- Run `/level-up`. The Three Ms interview surfaces one automation worth building. Build it. Wire a
  webhook or a schedule from `integrations/` so it runs without you.

## Week 3+
- Weekly `/level-up`. One shipped artifact per week. Stand up the dashboard so your work is visible.

You'll know it's working when: a teammate asks something and you'd rather ask your AIOS too; you stop
opening new tabs; you stop trying to remember business facts.
