# The Three Ms & The Four Cs

The two mental models behind the AI-OS. The Three Ms are how the operator thinks.
The Four Cs are how the system is built.

## The Three Ms — the operator's brain

| M | Question | In practice |
|---|----------|-------------|
| **Mindset** | "To what extent can AI leverage this?" | The default question before any new task. |
| **Method** | Constraint → EAD → Map → Autonomy → KPI | Find the bottleneck, then Eliminate / Automate / Delegate, map the process, set autonomy + a metric. |
| **Machine** | Build boring, reliable workflows | Dull and dependable beats clever and fragile. |

## The Four Cs — the architecture

| C | Layer | Pass test |
|---|-------|-----------|
| **Context** | AI knows your business | A fresh session answers basic questions about the project. |
| **Connections** | AI accesses live data | Calendar/tasks/files queryable without manual paste. |
| **Capabilities** | AI runs workflows | A short phrase triggers a multi-step process that ships an artifact. |
| **Cadence** | AI runs autonomously | Value delivered with the laptop closed. |

> [!tip] Build order
> Climb the Cs in order. Cadence needs Capabilities, which need Connections, which
> need Context. Don't skip down the stack. The lowest weak layer is always the priority.

## The three signals it's working

1. Team reaches out, but the AIOS answers faster and with exact sources. You are no
   longer the knowledge bottleneck.
2. Context-switching drops. Default for a new task is to ask the AIOS, not open five apps.
3. Knowledge leaves your head. You stop memorizing and trust retrieval.
